# Loopy Theory Cycle 20260520_114745

## Target

Define when a Loopy loop should continue autonomously and when it should ask the user.

## Claim

A Loopy loop should continue autonomously while the next move is clear.

## Evidence

- The user wants the skill to run loops, not stop after every note.
- `loopy-theory` already says to continue from `Next question` unless a stop condition is met.
- Autonomous continuation is useful when the next question follows directly from the previous cycle.
- Asking too often turns the loop back into manual step approval.

## Objection

"Next move is clear" can be too broad. An agent may keep going even when the direction is clear but the stakes are wrong, the evidence is blocked, or the artifact will change scope.

The loop needs a distinction between local uncertainty and authority uncertainty.

## Revision

A Loopy loop may continue autonomously when the next move stays inside the current target, uses an available verification surface, and cannot change user intent or artifact scope.

It should ask the user when:

- the target itself is ambiguous;
- the next move would change scope or priority;
- verification is blocked;
- multiple plausible directions depend on user preference;
- promotion would commit to a high-impact claim or artifact boundary the user has not accepted.

## Why It Is Stronger

The revision keeps autonomy without pretending the loop has authority over user intent.

It lets the agent keep moving on evidence-driven local questions, while stopping at decisions that need preference, scope, or authority.

## Progress Toward Target

This changes the decision boundary for Loopy loops.

Before: continue if a next question exists.

After: continue only when the next move is target-local, verifiable, and does not require user authority.

This is stable enough to promote into `THEORY_LOOPY.md`.
