# Loopy Theory Cycle 20260520_115244

## Target

Check whether the simplified `loopy-theory` skill still preserves enough structure for repeatable loops.

## Claim

The simplified skill is enough because the workflow now follows `Unstable -> Tested -> Stable`.

## Evidence

- The workflow is shorter and clearer.
- The final theory is now very compact.
- The output section still preserves cycle notes and final theory separately.

## Objection

The back half of the skill still uses older terms: target, current claim, revision, autonomous progress. Those are not wrong, but they make the skill feel like two theories stitched together.

If the skill is meant to improve itself while making theory, the state vocabulary should match the core loop.

## Revision

Use Loopy-native state terms:

- Unstable claim
- Rejection test
- Survivor
- Next weak point
- Decision

Roles should also follow the loop:

- Theorist drafts.
- Critic tests.
- Synthesizer keeps the survivor.
- Archivist records and chooses next.

## Why It Is Stronger

The revision makes the skill more internally consistent.

It also makes it easier to inspect later: every section now points back to the same core theory.

## Progress Toward Target

This improves repeatability without adding complexity.

The change should be applied to the skill, not promoted into the final theory because it is implementation policy.
