# Loopy Theory Cycle 20260520_114746

## Target

Define what Loopy must preserve between cycles so later loops can resume without rereading every rough note.

## Claim

Loopy should preserve the current artifact and the latest next question.

## Evidence

- `THEORY_LOOPY.md` is the clean artifact that represents the current stable theory.
- Cycle notes contain useful history, but rereading every note is inefficient and may pull failed claims back into the main theory.
- The next question is needed to continue the loop.

## Objection

Current artifact plus next question is too little. A later loop also needs to know what is still weak and what kind of verification is expected.

Without that, the loop may repeat old objections or promote results without checking the right surface.

## Revision

Loopy should preserve a compact loop state, separate from rough history:

- Current artifact: the latest promoted result.
- Open question: the next target-local weakness or uncertainty.
- Weak point: why the current artifact is not final.
- Verification surface: what evidence can change the artifact.
- Last decision: continue, promote, replace, stop, or ask.

Cycle notes preserve history. Final artifacts preserve stable claims. Loop state preserves resumability.

## Why It Is Stronger

The revision separates three things that should not be mixed:

- history;
- stable artifact;
- resumable state.

This makes Loopy more autonomous without forcing every future cycle to reread all prior reasoning.

## Progress Toward Target

This changes the minimum operational structure.

Before: Loopy needed target, state, move, verification, and decision.

After: Loopy also needs a compact resumable state that carries the current artifact, open question, weak point, verification surface, and last decision.

This is stable enough to promote.
