# Loopy Theory Cycle 20260520_113945

## Target

Define the minimum condition for a Loopy skill.

## Claim

A Loopy skill is a goal-driven loop: it repeatedly tests and improves a target claim or task until it can continue, stop, or change direction.

## Evidence

- A normal checklist can finish all listed steps while still failing the user's real goal.
- Autonomous work often changes after each step because new evidence, errors, or blockers appear.
- `loopy-theory` itself became clearer when it separated target, loop state, progress verification, and stop conditions.
- The user wants skills that proceed by following a loop toward a goal, not isolated one-shot procedures.

## Objection

"Goal-driven loop" is too broad. Many workflows have goals and repetition, but that alone does not make them useful for autonomous agents.

The missing condition is accountable progress. The loop must preserve what changed and decide the next move from evidence, not just repeat steps.

## Revision

A Loopy skill is a goal-driven loop that preserves state and forces an evidence-based next decision.

Minimum condition:

- Target: what the loop is trying to explain, distinguish, test, or improve.
- State: what is known, changed, blocked, or still weak.
- Move: the next smallest action that can change the state.
- Verification: evidence that the move changed the state or failed to.
- Decision: continue, promote, replace, stop, or ask.

## Why It Is Stronger

The revision separates Loopy from an ordinary checklist.

A checklist can say what to do. A Loopy skill must say what changed, how it was checked, and what decision follows.

This gives a boundary: if a skill has no state, verification, or next decision, it is not yet a Loopy skill.

## Progress Toward Target

This revision defines a usable minimum condition.

It changes the boundary from "loops toward a goal" to "loops through a target claim or task with preserved state and evidence-based decisions."

The claim is stable enough to promote as the first version of `THEORY_LOOPY.md`.
